# Core Product Schema (Source of Truth)

This document captures the canonical schema for product, stats, pricing, and stock-history tables.
**Rule: never recreate these tables — they hold live production data.**

All mutations are logged automatically by triggers. No hidden writes.

---

## `public.products` (59 columns)

Primary catalog table. Bilingual (sv/en), with structured attributes, SEO, inventory, and pricing.

### Identity
| column | type | notes |
|---|---|---|
| `id` | uuid PK | `gen_random_uuid()` |
| `handle` | text | URL slug, auto-generated by `generate_product_handle()` trigger |
| `vendor` | text | default `'4ThePeople'` |
| `status` | text | `'active'` default; controls lifecycle |
| `is_visible` | bool | shop visibility |
| `is_sellable` | bool | purchase eligibility |
| `display_order` | int | manual sort |
| `created_at`, `updated_at` | timestamptz | |

### Content (bilingual)
`title_sv` (NOT NULL), `title_en`, `description_sv/en`, `extended_description_sv/en`, `ingredients_sv/en`, `recipe_sv/en`, `feeling_sv/en`, `effects_sv/en`, `usage_sv/en`, `dosage_sv/en`, `variants_sv/en`, `storage_sv/en`, `safety_sv/en`, `hook_sv/en`.

### Media
`image_urls` (text[]), `badge` (text).

### Pricing & cost
| column | type | notes |
|---|---|---|
| `price` | numeric NOT NULL default 0 | guarded by `guard_product_price_range` (0–100000) |
| `original_price` | numeric | strike-through reference |
| `cost_price` | numeric | unit cost; drives margin & discount engine |
| `currency` | text NOT NULL default `'SEK'` | |

> **Trigger**: `log_product_price_change` writes to `price_history` whenever `price` OR `cost_price` changes.

### Inventory
| column | type | notes |
|---|---|---|
| `stock` | int NOT NULL default 0 | physical stock |
| `reserved_stock` | int NOT NULL default 0 | held during checkout |
| `allow_overselling` | bool NOT NULL default false | |
| `low_stock_threshold` | int NOT NULL default 5 | drives `low_stock` event |
| `restock_amount` | int NOT NULL default 50 | suggested PO size |
| `units_sold_7d`, `units_sold_30d` | int | velocity (rolling) |
| `last_sold_at` | timestamptz | |
| `weight_grams`, `package_weight_grams` | int | shipping math |
| `is_concentrate` | bool | category flag |

### Categorization & SEO
`category` (text legacy), `tags` (text[]), `certifications` (text[]), `specifications` (jsonb), `meta_title`, `meta_description`, `meta_keywords`.

### Triggers
- `generate_product_handle` (BEFORE INSERT) — slug from `title_sv`.
- `guard_product_price_range` (BEFORE INSERT/UPDATE) — blocks invalid prices, logs to `security_events`.
- `init_product_stats` (AFTER INSERT) — creates matching `product_stats` row.
- `log_product_price_change` (AFTER UPDATE) — appends to `price_history` on price/cost change.

## `public.product_stats`

1:1 with `products`. Aggregated counters; updated by app + triggers.

| column | type | notes |
|---|---|---|
| `product_id` | uuid PK FK→products | `ON DELETE CASCADE` |
| `views`, `cart_adds`, `cart_removes`, `purchases`, `units_sold` | bigint NOT NULL default 0 | counters |
| `revenue` | numeric(12,2) NOT NULL default 0 | lifetime gross |
| `last_viewed_at`, `last_purchased_at` | timestamptz | |
| `created_at`, `updated_at` | timestamptz | `update_updated_at_column()` trigger |

### RLS
- SELECT: public
- INSERT/UPDATE/DELETE: admins only (`is_admin(auth.uid())`)

## `public.price_history` (append-only audit log)

Every price OR cost change on `products` lands here automatically via the
`log_product_price_change()` trigger. **Never insert manually** outside of that trigger
unless backfilling — the trigger fires on UPDATE only.

| column | type | notes |
|---|---|---|
| `id` | uuid PK | |
| `product_id` | uuid FK→products | `ON DELETE CASCADE` |
| `old_price`, `new_price` | numeric(10,2) | |
| `old_cost`, `new_cost` | numeric(10,2) | |
| `change_reason` | text | optional human note |
| `source` | text NOT NULL default `'manual'` | `'manual'` \| `'system'` (set from `auth.role()`) |
| `changed_by` | uuid | `auth.uid()` at trigger time |
| `changed_at` | timestamptz NOT NULL default `now()` | |

Index: `(product_id, changed_at DESC)`.

### RLS
- SELECT: public
- INSERT/DELETE: admins only
- UPDATE: not allowed (audit log is immutable in practice)

## `public.stock_history` (added)

Mirrors `price_history` for stock + reserved_stock changes. Trigger-driven, append-only.

| column | type | notes |
|---|---|---|
| `id` | uuid PK | |
| `product_id` | uuid FK→products | `ON DELETE CASCADE` |
| `old_stock`, `new_stock` | int | |
| `old_reserved`, `new_reserved` | int | |
| `delta_stock` | int generated | `new_stock - old_stock` |
| `change_reason` | text | optional |
| `source` | text NOT NULL default `'manual'` | `'manual'` \| `'system'` (`auth.role()`-derived) |
| `changed_by` | uuid | `auth.uid()` |
| `changed_at` | timestamptz NOT NULL default `now()` | |

### Trigger
`log_product_stock_change()` (AFTER UPDATE on products) — fires when `stock` OR `reserved_stock` changes. Same shape as `log_product_price_change()`.

### RLS
Same as `price_history`: public read, admin insert/delete, no update.

## Logging contract

Every mutation that affects pricing or inventory is captured in:

| change | written to | by |
|---|---|---|
| `products.price` / `cost_price` | `price_history` | `log_product_price_change` trigger |
| `products.stock` / `reserved_stock` | `stock_history` | `log_product_stock_change` trigger |
| Out-of-range price | `security_events` | `guard_product_price_range` trigger |
| New product | `product_stats` (init row) | `init_product_stats` trigger |
| Pipeline stage decisions | `change_log` | `logChange()` (TS, `ecommerceOrchestrator`) |
| Deterministic events | `ecommerce_events` | `emit_ecommerce_event()` RPC |

**No silent writes.** Anything that bypasses these triggers is a bug.
